findid¶

Find the id(s) of the closest earthquake to input parameters.

To print the authoritative id of the event closest in time and space inside a 100 km, 16 second window to “2017-08-30 03:00:33 UTC 37.571 118.888”:

%(prog)s 2017-08-30T03:00:33 37.571 -118.888

To make a similar query but with the time shifted by 2 minutes, and a custom time window of 3 minutes:

%(prog)s -w 180 2017-08-30T03:00:33 37.571 -118.888

To print the authoritative id AND the url of the event closest in time and space to that point:

%(prog)s -u -w 180 2017-08-30T03:00:33 37.571 -118.888

To print all of the ids associated with the event closest to above:

%(prog)s -a 2015-03-29T23:48:31 -4.763 152.561

To print the id(s), time/distance deltas, and azimuth from input to nearest event:

%(prog)s -v 2015-03-29T23:48:31 -4.763 152.561

Notes:
  • The time format at the command line must be of the form “YYYY-MM-DDTHH:MM:SS”. The time format in an input csv file

can be either :YYYY-MM-DDTHH:MM:SS” OR “YYYY-MM-DD HH:MM:SS”. This is because on the command line the argument parser would be confused by the space between the date and the time, whereas in the csv file the input files are being split by commas. - Supplying the -a option with the -f option has no effect.

usage: findid [-h] [-r RADIUS] [-w WINDOW] [-a] [-u] [-v] time lat lon

Positional Arguments¶

time Time of earthquake, formatted as YYYY-mm-dd or YYYY-mm-ddTHH:MM:SS
lat Latitude of earthquake
lon Longitude of earthquake

Named Arguments¶

-r, --radius Change search radius from default of 100 km.
-w, --window Change time window of 16 seconds.
-a, --all

Print all ids associated with event.

Default: False

-u, --url

Print URL associated with event.

Default: False

-v, --verbose

Print time/distance deltas, and azimuth from input parameters to event.

Default: False
