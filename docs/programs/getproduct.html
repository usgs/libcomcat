getproduct¶

Download product content files from USGS ComCat.

To download ShakeMap grid.xml files for a box around New Zealand during 2013:

%(prog)s shakemap “grid.xml” -o /home/user/newzealand -b 163.213 -178.945 -48.980 -32.324 -s 2013-01-01 -e 2014-01-01

Note that when specifying a search box that crosses the -180/180 meridian, you simply specify longitudes as you would if you were not crossing that meridian.

Note: Some product content files do not always have the same name, usually because they incorporate the event ID into the file name, such as with most of the files associated with the finite-fault product. To download these files, you will need to input a unique fragment of the file name that can be matched in a search.

For example, to retrieve all of the coulomb input files for the finite-fault product, you would construct your search like this: %(prog)s finite-fault _coulomb.inp -o ~/tmp/chile -b -76.509 -49.804 -67.72 -17.427 -s 2007-01-01 -e 2016-05-01 -m 6.5 9.9

To retrieve the moment rate function files, do this: %(prog)s finite-fault .mr -o ~/tmp/chile -b -76.509 -49.804 -67.72 -17.427 -s 2007-01-01 -e 2016-05-01 -m 6.5 9.9

usage: getproduct [-h] [-o OUTPUTFOLDER] [-b lonmin lonmax latmin latmax]
                  [-r lat lon rmax] [-s STARTTIME] [-e ENDTIME] [-a AFTER]
                  [-m minmag maxmag] [-c CATALOG] [-n CONTRIBUTOR]
                  [-i EVENTID] [-p PRODUCTPROPERTIES] [-t EVENTPROPERTIES]
                  [-l] [--get-version {first,last,all}] [--get-source SOURCE]
                  PRODUCT [CONTENTLIST [CONTENTLIST ...]]

Positional Arguments¶

PRODUCT The name of the desired product (shakemap, dyfi, etc.)
CONTENTLIST The names of the product contents (grid.xml, stationlist.txt, etc.)

Named Arguments¶

-o, --output-folder
 

Folder where output files should be written (must exist).

Default: “/Users/mhearne/src/python/libcomcat”

-b, --bounds Bounds to constrain event search [lonmin lonmax latmin latmax]
-r, --radius Search radius in KM (use instead of bounding box)
-s, --start-time
 Start time for search (defaults to ~30 days ago). YYYY-mm-dd or YYYY-mm-ddTHH:MM:SS
-e, --end-time End time for search (defaults to current date/time). YYYY-mm-dd or YYYY-mm-ddTHH:MM:SS
-a, --after Limit to events after specified time. YYYY-mm-dd or YYYY-mm-ddTHH:MM:SS
-m, --mag-range
 Min/max magnitude to restrict search.
-c, --catalog Source catalog from which products derive (atlas, centennial, etc.)
-n, --contributor
 Source contributor (who loaded product) (us, nc, etc.)
-i, --event-id Event ID from which to download product contents.
-p, --product-property
 Product property (reviewstatus:approved).
-t, --event-property
 Event property (alert:yellow, status:REVIEWED, etc.).
-l, --list-url

Only list urls for contents in events that match criteria.

Default: False

--get-version

Possible choices: first, last, all

Get contents for first, last or all versions of event.

Default: “last”

--get-source

Get contents for the “preferred” source, “all” sources, or a specific source (“us”).

Default: “preferred”
