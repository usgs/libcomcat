getphases¶

Download phase data for matching events into CSV or Excel format.

The resulting files will contain a “header” consisting of basic earthquake information (Id,Time,Latitude,Longitude,Depth,Magnitude, etc.) at the top of the file. In the Excel format, this information takes the first several rows and two columns. In CSV format, the values are the first lines of the file, prepended with a “#” (comment) character.

To download phase data to Excel format for a small rectangle in Oklahoma in 2017: %(prog)s ~/tmp/phase_data -b -97.573 -97.460 36.247 36.329 -s 2017-08-26 -e 2017-09-15 -f excel

To download phase data for the 2017 7.1 Mexico City event: %(prog)s ~/tmp/phase_data -i us2000ar20 -f excel

usage: getphases [-h] [-b lonmin lonmax latmin latmax] [-r lat lon rmax]
                 [-s STARTTIME] [-e ENDTIME] [-i EVENTID] [-t AFTER]
                 [-m minmag maxmag] [-c CATALOG] [-n CONTRIBUTOR] [-f FORMAT]
                 DIRECTORY

Positional Arguments¶

DIRECTORY Output directory.

Named Arguments¶

-b, --bounds Bounds to constrain event search [lonmin lonmax latmin latmax]
-r, --radius Search radius in KM (use instead of bounding box)
-s, --start-time
 Start time for search (defaults to ~30 days ago). YYYY-mm-dd, YYYY-mm-ddTHH:MM:SS, or YYYY-mm-ddTHH:MM:SS.s
-e, --end-time End time for search (defaults to current date/time). YYYY-mm-dd, YYYY-mm-ddTHH:MM:SS, or YYYY-mm-ddTHH:MM:SS.s
-i, --event-id Extract phase data for a single event, using ComCat event ID.
-t, --time-after
 Limit to events after specified time. YYYY-mm-dd or YYYY-mm-ddTHH:MM:SS
-m, --mag-range
 Min/max (authoritative) magnitude to restrict search.
-c, --catalog Source catalog from which products derive (atlas, centennial, etc.)
-n, --contributor
 Source contributor (who loaded product) (us, nc, etc.)
-f, --format

Possible choices: csv, tab, excel

Output format.

Default: “csv”
